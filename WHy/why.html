But firstly let`s describe the DNS-based load balancing.

DNS-Based Load Balancing

DNS-based load balancing is a specific type of load balancing that uses the DNS to distribute traffic across several servers. It does this by providing different IP addresses in response to DNS queries. Load balancers can use various methods or rules for choosing which IP address to share in response to a DNS query. and round-robin DNS is the most used one.

Round-Robin DNS

Round-robin DNS, like other DNS-based load balancing methods, aims to improve a site's performance and reliability by distributing traffic. Round-robin DNS, on the other hand, uses a type of DNS server called an authoritative nameserver to perform load balancing instead of a specialized software- or hardware-based load balancer.

DNS records called A records or AAAA records, which contain a domain's name and IP address, are held by authoritative nameservers.

The goal of a DNS query submitted by a client is to locate the A (or AAAA) record. In a basic setup, a domain will have a single A record associated with a single IP address, implying that a DNS query will always return the same IP address.

Domains with round-robin DNS, on the other hand, have multiple A records, each of which is associated with a different IP address. IP addresses rotate in a round-robin fashion as DNS queries arrive, spreading the requests across the associated servers.

If round-robin DNS has five IP addresses, a DNS query will only return the first IP address for every sixth request. This setup reduces each server's workload, making it less likely to become overwhelmed by requests because each IP address corresponds to a different server.
